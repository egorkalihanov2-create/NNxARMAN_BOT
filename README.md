# Telegram text game bot

Каркас Telegram-бота для интерактивной текстовой игры.

## Что уже заложено

- Игроки, прогресс, события и загрузки хранятся в Supabase.
- Сюжет читается из внешнего YAML-файла.
- Поддержаны сегменты с выбором, текстовым вводом и загрузкой изображения.
- Для файлов есть лимит по размеру: по умолчанию 10 МБ.
- Фото/файлы игроков сохраняются в приватный Supabase Storage bucket.

Подробное решение по хранению: `docs/storage_plan.md`.
Формат сценария и персонажных реплик: `docs/story_format.md`.
Проверка на Railway: `docs/deploy_railway.md`.

## Быстрый запуск

1. Создать проект Supabase.
2. Выполнить SQL из `supabase/schema.sql`.
3. Создать приватный Storage bucket, например `player-uploads`.
4. Скопировать `.env.example` в `.env` и заполнить значения.
5. Установить Python 3.12+.
6. Установить зависимости:

```bash
pip install -r requirements.txt
```

7. Запустить бота:

```bash
python -m src.main
```

На Windows также может использоваться:

```bash
py -m src.main
```

Если команда `python` открывает Microsoft Store или не запускает проект, используйте полный путь к установленному Python:

```powershell
& "$env:LOCALAPPDATA\Programs\Python\Python312\python.exe" -m src.main
```

## Где менять историю

Основной файл контента: `content/story.yaml`.

Сейчас там лежит короткий пример с тремя типами интерактива:

- `choice` - выбор кнопкой.
- `text_input` - проверка текстового ответа.
- `upload` - загрузка PNG/JPEG до 10 МБ.

## Важное про фото для зина

Если важно получить оригинальное качество, лучше просить игроков отправлять изображение как файл. Обычная отправка как фото может сжиматься Telegram.
